# 캘린더 기반 Todo 웹서비스 MVP PRD

## 🎯 핵심 정보

**목적**: 모두의 할일이 하나의 공개 캘린더에 모여 표시되고, 로그인한 본인만 자신의 할일을 등록·수정·삭제할 수 있는 캘린더형 할일 관리 서비스
**사용자**: 자신의 일정을 캘린더 형태로 정리하고, 동시에 다른 사람들의 공개 일정도 함께 열람하고 싶은 개인 사용자

## 🚶 사용자 여정

### 여정 A. 할일 열람 및 작성 (핵심 플로우)

```text
1. 캘린더 페이지 (홈)
   ↓ 날짜 셀의 "+" 클릭

2. 로그인 여부 체크
   ↓

   [비로그인] → 로그인 페이지로 이동
   [로그인 상태] → 할일 작성 드로어 오픈
   ↓

3. 할일 작성 드로어에서 제목·본문(에디터)·이미지 작성 후 저장
   ↓ 저장 완료

4. 캘린더 페이지로 복귀 → 해당 날짜 셀에 할일 pill 표시
```

### 여정 B. 기존 할일 조회 및 수정

```text
1. 캘린더 페이지 (홈)
   ↓ 날짜 셀의 할일 pill 클릭

2. 할일 상세/수정 드로어 오픈 (상세 내용 lazy-load)
   ↓

   [작성자 본인] → 수정/삭제 버튼 노출 → 수정 저장 또는 삭제
   [타인의 글] → 읽기 전용으로 내용만 열람
   ↓

3. 완료 → 드로어 닫힘 → 캘린더 페이지로 복귀
```

### 여정 C. 회원가입 및 로그인

```text
1. 캘린더 페이지 (홈)
   ↓ 헤더의 로그인 버튼 클릭

2. 로그인 페이지
   ↓

   [계정 없음] → 회원가입 페이지 → 이메일 인증 메일 발송 → 인증 완료 후 로그인 페이지로 복귀
   [계정 있음] → 이메일/비밀번호 입력 → 로그인
   ↓

3. 로그인 성공 → 캘린더 페이지로 복귀 (헤더에 사용자 메뉴 노출)
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|---|---|---|---|---|
| **[F001]** | 캘린더 월간 조회 | 날짜별 할일 제목(pill)을 표시. PC·태블릿은 월간 그리드, 모바일 폭에서는 날짜별 카드 리스트로 반응형 전환 | 서비스의 핵심 화면이자 모든 흐름의 시작점 | 캘린더 페이지 |
| **[F002]** | 할일 상세 조회 | 할일 pill 클릭 시 제목·본문·작성자를 드로어에서 열람 | 공개 조회가 서비스의 핵심 전제 | 캘린더 페이지, 할일 상세/수정 드로어 |
| **[F003]** | 할일 등록 | 날짜를 선택해 제목·본문·이미지를 작성하고 저장 | 서비스의 존재 목적 그 자체 | 할일 작성 드로어 |
| **[F004]** | 할일 수정 | 본인이 작성한 할일의 제목·본문·이미지를 수정 | 잘못 등록한 정보를 고칠 최소 수단 | 할일 상세/수정 드로어 |
| **[F005]** | 할일 삭제 | 본인이 작성한 할일을 삭제 | 불필요해진 일정을 정리할 최소 수단 | 할일 상세/수정 드로어 |
| **[F006]** | 위지윅 서식 작성 | 굵게·목록·제목 등 서식이 적용된 본문 작성 | 단순 텍스트만으로는 일정 설명력이 부족 | 할일 작성 드로어, 할일 상세/수정 드로어 |
| **[F007]** | 이미지 첨부 | 본문 작성 중 이미지를 드래그·붙여넣기로 삽입 | 일정에 사진 첨부는 캘린더 서비스의 기본 니즈 | 할일 작성 드로어, 할일 상세/수정 드로어 |
| **[F008]** | 월 이동 네비게이션 | 이전/다음 달로 캘린더 화면을 이동 | 캘린더 서비스로서 최소한의 탐색 기능 | 캘린더 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|---|---|---|---|---|
| **[F010]** | 기본 인증 | 회원가입/로그인/로그아웃만 제공 | 소유자 판별 없이는 쓰기 권한 제어가 불가능 | 로그인 페이지, 회원가입 페이지 |
| **[F011]** | 작성자 표시 | 할일 pill·상세에 작성자 닉네임을 표시 | 공개 캘린더에서 누구의 일정인지 구분 필요 | 캘린더 페이지, 할일 상세/수정 드로어 |

- 프로필 상세 관리 (아바타, 자기소개 등)

## 📄 페이지별 상세 기능

### 캘린더 페이지 (홈)

> **구현 기능:** `F001`, `F002`, `F008`, `F011` | **메뉴 위치:** 홈

| 항목 | 내용 |
|---|---|
| **역할** | 전체 사용자의 할일을 월간 그리드로 보여주는 서비스의 메인 화면 |
| **진입 경로** | 헤더 로고/홈 메뉴 클릭, 또는 서비스 최초 접속 시 기본 진입 |
| **사용자 행동** | 월을 이동하며 날짜별 할일 목록을 확인하고, 빈 셀 또는 기존 pill을 클릭 |
| **주요 기능** | • PC·태블릿(768px 이상): 월간 그리드에 날짜별 할일 pill 표시(제목+작성자)<br>• 모바일(768px 미만): 이번 달 날짜별 카드 리스트로 전환, 카드마다 할일 목록과 추가 버튼 노출<br>• 이전/다음 달 이동<br>• 빈 셀/카드 "+" 클릭<br>• 기존 pill 클릭 |
| **다음 이동** | 빈 셀 "+" 클릭 시(로그인) → 할일 작성 드로어, (비로그인) → 로그인 페이지 / pill 클릭 → 할일 상세/수정 드로어 |

## 🗄️ 데이터 모델

### profiles (공개 프로필)

| 필드 | 설명 | 타입/관계 |
|---|---|---|
| id | 고유 식별자(인증 계정과 1:1) | UUID |
| display_name | 화면에 노출되는 닉네임 | text |
| created_at | 프로필 생성 시각 | timestamptz |

### todos (할일)

| 필드 | 설명 | 타입/관계 |
|---|---|---|
| id | 고유 식별자 | UUID |
| user_id | 작성자 | → profiles.id |
| title | 할일 제목(캘린더 pill에 노출) | text |
| content | 위지윅 에디터 본문 | jsonb |
| date | 할일이 표시될 날짜 | date |
| image_paths | 본문에 삽입된 이미지 경로 목록 | text[] |

## 🛠️ 기술 스택 (최신 버전)

### 🎨 프론트엔드 프레임워크

- **Next.js 16** (App Router) - React 풀스택 프레임워크
- **TypeScript** - 타입 안전성 보장
- **React 19** - UI 라이브러리

### 🎨 스타일링 & UI

- **TailwindCSS v4** (설정파일 없는 새로운 엔진) - 유틸리티 CSS 프레임워크
- **shadcn/ui** (`radix-nova` 스타일) - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리

### 📝 폼 & 검증

- **React Hook Form** - 폼 상태 관리
